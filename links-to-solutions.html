Solutions

There are two links in front of each task description. The examine link points to the Github commit containing the solution. The download link points to a .zip archive that contains the source code.

Virtual Socket

  1. [Examine] [Download] (2p) Make an application for testing purposes, e.g. client sends data to server and server displays it on screen. It is enough for the application to send data only in one direction (and later acknowledgements in the other). Use the test application to test what you implement later.
  2. [Examine] [Download] (2p) Make a virtual socket that randomly drops a packet (i.e. does not pass it to UDP socket or to the application)
  3. [Examine] [Download] (3p) Add to the virtual socket: randomly delays a packet, before passing it forward
  4. [Examine] [Download] (5p) Add to the virtual socket: randomly generates bit errors on the data of the packet, a single bit error is enough.

Positive and Negative ACKs

  1. [Examine] [Download] (2p) Add to your data packet some method for detecting at least a single bit error.
  2. [Examine] [Download] (2p) Reliable data transfer with positive and negative ACKs
  3. [Examine] [Download] (4p) Reliable data transfer with only positive ACKs
  4. [Examine] [Download] (4p) Reliable data transfer with only negative ACKs

Reliable Transfer Protocol

  1. [Examine] [Download] (12p) Implement reliable data transfer on top of UDP that can handle packet loss, packet delay and single bit error (detect error and react accordingly).
